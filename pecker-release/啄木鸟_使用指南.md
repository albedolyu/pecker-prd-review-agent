# 啄木鸟 (Pecker) — 使用指南

> 啄木鸟专门在树上找虫。PRD 就是那棵树，bug 就是虫。
> 十只鸟各司其职：评审团并行找虫，苍鹰交叉校验，杜鹃验证质量，鸮鹦整理知识库，信鸽从下游带回反馈，伯劳审核产出。

## 前置条件

- Python 3.10+
- Anthropic API Key（支持中转站）

## 1. 安装依赖

```bash
pip install -r requirements.txt
```

## 2. 克隆共享知识库

共享知识库由 PM 统一维护，所有人评审时读写同一份：

```bash
git clone https://github.com/xinshu001/prd-wiki.git
```

记下克隆到的本地路径，下一步填入 `.env`。

## 3. 配置 .env

复制 `.env.example` 为 `.env`，填入以下内容：

```bash
# 必填：API Key
ANTHROPIC_API_KEY=向 PM 索取

# 必填：中转站地址
ANTHROPIC_BASE_URL=https://pikachu.claudecode.love

# 必填：共享知识库本地路径（第 2 步克隆的路径）
WIKI_PATH=填入你的本地路径

# 必填：你的名字
REVIEWER=你的名字

# 可选：飞书通知
FEISHU_WEBHOOK=
```

## 4. 运行 PRD 评审

```bash
# 自动路由（推荐）— Haiku 判断 PRD 复杂度，自动选模型
# 默认开启并行 Workers（织布鸟+猫头鹰+渡鸦+鸬鹚）
python run_session.py 搜索优化 --workspace /path/to/workspace

# 指定评审人
python run_session.py 搜索优化 --reviewer zhangsan --workspace ./my-project

# 强制指定模型
python run_session.py 搜索优化 --model opus     # 深度评审
python run_session.py 搜索优化 --model sonnet   # 标准评审
python run_session.py 搜索优化 --model haiku    # 格式检查

# 关闭并行评审（单 agent 串行模式）
python run_session.py 搜索优化 --no-parallel
```

**评审前请确保：**
- 业务资料已放入 **`raw/`** 目录
- 待评审 PRD 已放入 **`prd/`** 目录

## 5. 会话内命令

| 命令 | 说明 |
|------|------|
| 直接输入文字 | 与啄木鸟对话，推进评审流程 |
| **`/submit`** | 评审完成后执行 — git add → commit → push |
| `/flock` | 显示鸟类全家福 |
| `/fortune` | 随机一条啄木鸟金句 |
| `/stats` | 查看成就列表 |
| `/nest` | 查看知识森林状态 |
| `/blame` | 查看最近的漏报信号 |
| `exit` / `q` | 退出会话 |

## 6. 典型评审流程

```
启动 session — 啄木鸟上线（随机 ASCII Art）
    │
    ▼
Phase 0: 检查工作目录结构
    │
    ▼
Phase 0.5: 知识库预检 — 检查覆盖情况，报告知识盲区
    │          → 如有盲区，补充资料到 raw/ 再继续
    ▼
Phase 1: ingest raw/ 资料 → wiki 知识页
    │
    ▼
Phase 2: 并行评审（默认开启）
    │    织布鸟 → 结构层  |  猫头鹰 → 质量层
    │    渡鸦   → AI Coding |  鸬鹚  → 数据质量
    │    → 合并去重 → 依据验证 → 啄伤度评分
    │
    ▼
Phase 2.5: 苍鹰交叉校验（可选，手动触发）
    │    → 误报检测 / 漏报补充 / 冲突调解
    ▼
Phase 3: 逐条确认改进项
    │    → Y 接受 / N 驳回 / E 修改 / + 新增
    ▼
Phase 4: 生成报告 + 知识库反哺 + 成就解锁
    │
    ▼
/submit → git push
    │
    ▼
伯劳审核产出（手动触发）
```

## 7. 安全机制

| 保护 | 说明 |
|------|------|
| **文件权限** | `raw/`、`prd/`、`review-rules/` 只读 |
| **bash 白名单** | 只允许 git 和 pwd 命令 |
| **危险命令拦截** | `rm -rf`、`git push -f`、`curl` 等直接拒绝 |
| **push 确认** | `git push` 需要手动确认 y/n |
| **路径围栏** | 所有文件操作限制在 workspace 内 |

## 8. 其他鸟类工具

### 伯劳 — PR 审核

```bash
python shrike_review.py --workspace ./workspace
python shrike_review.py --workspace ./workspace --output review_report.md
```

检查：安全红线、wiki 一致性（断链/命名/frontmatter）、报告完整性。

### 杜鹃 — 评审质量验证

```bash
# 从已有报告生成测试用例
python cuckoo_eval.py --report output/PRD_改动报告_20260411.md --generate-test-case

# 完整评测（需要测试用例）
python cuckoo_eval.py --report output/PRD_改动报告_20260411.md --test-case test_case.json

# 仅依据验证
python cuckoo_eval.py --report output/PRD_改动报告_20260411.md --workspace ./workspace
```

### 鸮鹦 — Wiki 知识库维护

```bash
python kakapo_dream.py --wiki-path /path/to/wiki               # 完整整理
python kakapo_dream.py --wiki-path /path/to/wiki --scan-only    # 只扫描
python kakapo_dream.py --wiki-path /path/to/wiki --dry-run      # 预览修复
python kakapo_dream.py --wiki-path /path/to/wiki --rebuild-index
```

### 苍鹰 — 交叉校验

```bash
python goshawk_advisor.py --prd prd/xxx.md --report output/PRD_改动报告_xxx.md --model opus
```

### 信鸽 — 反馈闭环

```bash
# AI Coding 完成后运行
python feedback.py --code-dir /path/to/code --report output/PRD_改动报告_xxx.md --prd prd/xxx.md
```

## 9. 多人协作

每个评审者在独立分支上工作：

```bash
python run_session.py 搜索优化 --reviewer pm1 --workspace ./shared-repo
python run_session.py 风险预警 --reviewer pm2 --workspace ./shared-repo
```

分支命名规则：`review/{reviewer}/{prd_name}/{日期}`

知识库通过 Git 自动同步：评审开始前 `git pull`，结束后 `git push`。

## 12. 故障排查

| 问题 | 解决方法 |
|------|---------|
| `ANTHROPIC_API_KEY` 报错 | 在 `.env` 中填入 key |
| `INVALID_API_KEY` 401 | 清掉环境中的 `ANTHROPIC_AUTH_TOKEN`（Claude Code 会注入旧 token） |
| `工作目录不存在` | 用 `--workspace` 指定正确路径 |
| `[blocked]` 提示 | 操作被权限围栏拦截 |
| 评审中途崩溃 | 重新运行同名 PRD，选择恢复 session |
| git push 失败 | 检查网络和 Git 凭证 |
| 飞书通知没收到 | 检查 FEISHU_WEBHOOK 是否正确 |
