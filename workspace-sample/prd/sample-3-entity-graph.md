# 企业关联图谱 · PRD v0.3

> Sample PRD · 长版 3500+ 字 · 故意埋了 4 维度完整评审点 · 测 e2e 完整流

## 1. 背景

### 1.1 业务背景

C 端用户使用我方企业查询产品时 · 除了基础工商信息 · 越来越关注"**关联关系**"。以下三类场景占用户 query 的 18%:

- **合同背调场景**:签合同前想看对方公司的实控人是谁 · 避免合同绕不过关键关系人
- **投资尽调场景**:VC / 个人投资人要看目标公司的股东 · 子公司 · 兄弟公司结构
- **反欺诈场景**:判断公司是否是"空壳"或"壳中套壳"(多层股权穿透后实控人是自然人还是机构)

当前版本只有**平面企业列表** · 无法展示**关联关系图谱**。用户需要手工拼接股东 / 子公司 / 对外投资三份数据 · 体验非常差。

### 1.2 数据背景

我方现有数据源:
- `dim_enterprise`:企业维度表(3000 万家)· T+1 更新
- `dwd_shareholder`:股东明细表(1.2 亿条)· T+1 更新
- `dwd_investment`:对外投资明细(0.8 亿条)· T+1 更新
- `dwd_legal_rep`:法人 · 监事 · 董事明细(1.5 亿条)· **实时更新**

### 1.3 竞品情况

企查查的"股权穿透图"、天眼查的"图谱"均已上线 · 日活贡献占比 ~15%。我方此功能上线后预期能追平。

## 2. 目标

提升用户深度使用 · 让用户从"查一次"变为"查一次挖三层"。同时为 VIP 订阅转化提供新的卖点(穿透层数限制)。

## 3. 用户场景

### 场景 A · 合同背调(高频)

张三准备和 B 公司签 500 万合同 · 打开 Pecker 搜索 B 公司 · 点"关联图谱"Tab · 看 B 公司的股东结构 · 一层穿透发现实控人是某自然人 · 点该自然人 · 跳转到"该人名下其他公司"· 确认实控人没有负面历史。

### 场景 B · 投资尽调(中频)

李四是个人投资人 · 要投 C 公司。用关联图谱查看 C 公司的对外投资 · 看 C 公司投过的项目质量如何 · 以此判断 C 公司管理团队的投资眼光。

### 场景 C · 反欺诈(低频但价值高)

王五准备和 D 公司合作 · 用关联图谱穿透 3 层 · 发现 D 公司的实控人同时控制 5 家被列入失信被执行人名单的公司 · 立即拉黑 D 公司。

## 4. 功能要求

### 4.1 图谱视图

企业详情页新增"**关联图谱**"Tab · 点击后加载图谱视图:

- **中心节点**:当前查询企业 · 圆形 · 最大 · 加粗边框
- **一层关系**(默认展开):
  - **股东**(向上):该企业的股东(自然人 / 法人)
  - **子公司**(向下):该企业控股 >50% 的子公司
  - **对外投资**(向右):该企业持股 <50% 的投资标的
  - **关联交易**(向左 · 有关联交易数据时显示):关联方
- **节点大小**:按注册资本比例缩放
- **节点颜色**:按企业状态
  - 绿色:存续
  - 红色:吊销
  - 灰色:注销

### 4.2 穿透

- 点击任一节点 → 以该节点为中心重新绘制图谱(双击效果)
- 支持"**穿透到实控人**"按钮 · 自动向上追溯 · 最多 3 层 · 直到找到实控人

### 4.3 筛选

图谱右侧固定栏:

- **按持股比例筛选**: > 5% / > 10% / > 20% / > 50%
- **按企业状态筛选**: 存续 / 吊销 / 注销 / 迁出
- **按地域筛选**: 按省份
- **按行业筛选**: 按国标行业一级分类

### 4.4 导出

支持三种导出:
- **PDF**:含图谱 + 节点明细表
- **PNG**:高清图谱
- **XLSX**:节点 + 边的明细表

### 4.5 性能

- 首次加载 < 3 秒
- 节点数上限 100 个 · 超过自动聚合
- 边数上限 300 条

## 5. 数据

### 5.1 新增表

`enterprise_relations`(预计算关联关系 · 避免每次查询都实时算):

- `from_enterprise_id`
- `to_enterprise_id`
- `relation_type`:`shareholder` / `subsidiary` / `investment` / `related_party`
- `weight`:持股比例或投资金额
- `updated_at`

**更新频率**:离线 T+1 计算 · 结果 cache 到 Redis(TTL 24h)· 前端优先读 cache。

### 5.2 字段映射

图谱节点属性来源:
- `dim_enterprise.name`
- `dim_enterprise.registered_capital`
- `dim_enterprise.status`
- `dim_enterprise.legal_representative`
- `user_level` 字段判断是否限制穿透层数

### 5.3 数据来源口径

- 股东信息:`dwd_shareholder` 表(**T+1 离线更新**)
- 对外投资:`dwd_investment` 表(**T+1 离线更新**)
- 法人信息:`dwd_legal_rep` 表(**实时更新**)
- 企业状态:`dim_enterprise.status`(**实时更新**)

## 6. 依赖

### 6.1 内部服务依赖

- `enterprise_service`:查询基础企业信息(现有)
- `relation_compute_service`:计算关联关系图谱(**本次新建**)
- `search_service`:图谱内节点搜索(现有)

### 6.2 外部依赖

- **图可视化库**:D3.js 或 AntV G6(二选一 · 待技术 review)
- **图数据库**:Neo4j(用于 3+ 层穿透查询加速)

## 7. 交互细节

### 7.1 Loading

图谱加载时显示**骨架屏** · 3 秒内加载完。超过 5 秒显示"加载较慢 · 稍等"提示。

### 7.2 节点 hover

鼠标悬停节点时:
- 高亮该节点 + 所有直连边
- 显示悬浮卡片:企业名 · 法人 · 注册资本 · 状态

### 7.3 节点点击

- **单击**:跳转到该企业的详情页(在新 tab 打开)
- **双击**:以该节点为中心重新绘制图谱

### 7.4 图谱缩放

- 鼠标滚轮缩放
- 双指缩放(移动端 / 触控板)
- 点击"重置视图"回到初始布局

### 7.5 文案

- 穿透按钮:"**穿透到实控人**"
- 免费用户遇到穿透限制时:"穿透层数已达上限 · 升级 VIP 可查看更多"

## 8. 权限

- **免费用户**:最多查看 1 层关系 · 导出仅支持 PDF
- **VIP 用户**:最多 3 层穿透 · 支持全部导出格式
- **企业用户**:最多 5 层穿透 · 支持 API 导出

## 9. 数据合规

### 9.1 展示脱敏

对自然人股东/法人:
- 身份证号:默认隐藏 · 点击展示后 4 位
- 手机号:**不展示**
- 住址:只展示省市级 · 不到区

### 9.2 用户协议

首次打开"关联图谱"功能 · 弹窗让用户同意《企业关联图谱使用协议》。同意后记录在 `user_agreements` 表。

## 10. 指标

- MAU(本功能月活跃用户)
- 点击"关联图谱"Tab 的用户比例(相对进入企业详情页的用户)
- 平均穿透层数
- VIP 转化率(从免费点穿透限制弹窗到升级 VIP 的比例)

## 11. 时间排期

- 第 1-2 周:数据预计算 + 关系接口开发
- 第 3-4 周:前端图谱渲染 + 交互实现
- 第 5 周:内部联调 + 测试
- 第 6 周:灰度 10% 用户
- 第 7 周:全量上线

## 12. 风险

- 图数据库 Neo4j 我们没有运维经验 · 可能上线后出性能问题
- 关联数据精度依赖 T+1 计算 · 有延迟 · 可能出现"今天注销的公司还在图里显示为存续"的问题
