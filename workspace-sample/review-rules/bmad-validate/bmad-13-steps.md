# BMAD 13 步验证工作流

基于 BMAD PRD 验证框架，对 PRD 进行 13 步系统化验证。

## V-01 文档版本管理
检查文档是否有版本历史表、修订人、修订日期。

## V-02 格式规范性
验证 PRD 是否遵循标准模板（标准/变体/Legacy）。

## V-03 信息密度
检测对话式填充、冗余表达、重复短语。反模式示例：
- `"The system will allow users to..."` → 直接写功能
- `"Due to the fact that"` → `"because"`

## V-04 Brief 覆盖率
检查 Brief → PRD 的映射完整性，识别未覆盖的 Brief 要求。

## V-05 信息完整性
验证 PRD 中引用的所有信息（字段、规则、布局）能在文档内自洽，不依赖外部未注明的文档。

## V-06 成功标准可量化性
成功标准必须有具体数字或可测量的指标，不得使用模糊表述（如"提升体验"）。

## V-07 实现泄漏检测
功能需求（FR）不应含技术实现细节，如具体 API、框架名、数据库语句。

## V-08 领域合规性
检查是否涉及行业特有的合规要求（数据隐私、资质认证等）。

## V-09 SMART 验证
需求必须满足 Specific / Measurable / Attainable / Relevant / Traceable 五项标准。

## V-10 可追溯链完整性
愿景 → FR → 用户故事的链路不能断；每条 FR 必须能追溯到业务目标。

## V-11 边界与排除项
PRD 必须明确 Out of Scope，防止范围蔓延。

## V-12 整体质量评估
综合评估文档的完整性、一致性、可执行性。

## V-13 AI Coding 友好度
检查 PRD 是否足够支撑 AI Coding 工具（如 Copilot、Cursor）直接生成高质量实现，
包括：技术约定节、四态 UI、字段 DDL 可追溯性、伪代码覆盖。
