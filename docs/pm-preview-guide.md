# Pecker v8 · 内测指南

> 给 PM 同事看的一页纸 · 5 分钟上手

---

## 这是什么

Pecker 是帮 PM 做 PRD 评审的 AI 工具:
- **4 位 worker 鸟**并行审稿(业务 / 数据 / 体验 / 风险)
- **1 位苍鹰**交叉校验 · 撤回证据不足的 · 补上漏掉的
- **10 分钟**出一份可追溯的评审报告 · 可导出 .md / wiki / 飞书

## ⚠️ 内测期间 3 条铁律

1. **只用 sample PRD** · 不传真实业务 PRD(数据风险 · 见底部说明)
2. **不转发链接给组外同事** · 仅本群 N 位试用
3. **反馈随时群里发** · 或填文档结尾的 5 个问题

## 走一遍流程(5-8 分钟)

### Phase 0 · 上传 PRD

![Phase 0 截图位 · 你自己截]

- **Workspace**:选 `workspace-sample`
- **PRD**:拖一份 [sample PRD](#3-份-sample-prd) 进来 · 或贴粘贴
- **模式**:选"**简审**"(5 分钟) · 不用"精审"(10 分钟)
- **备注**:可选 · 填"内测试用"

点"**下一步 · 盲区预检 →**"

### Phase 1 · 盲区预检

自动跑 10-15 秒 · 扫 wiki · 识别**强相关 / 弱相关 / 知识盲区**三列。

点"**下一步 · 开始评审 →**"

### Phase 2 · 评审进行中(最值得看的一屏)

视觉上会看到:
- **4 个 worker 卡片**(业务 / 数据 / 体验 / 风险)并行跑
- **下方苍鹰卡片** · 等 worker 完成后交叉校验
- **SVG 依赖边**(dash-flow 动画)连接 4 worker 到苍鹰
- **底部深色 console** · 实时流式日志

大约 3-5 分钟跑完。

### Phase 1.5 · 运行质量检查(必经节点)

Phase 2 跑完 **不会自动跳** · 先停在"运行质量检查":
- session 分类:理想是 `productive` · 有问题会 `partial_silent` · 强制你选重跑或继续
- effective_consistency 环形:理想 > 90%
- 5 色失败矩阵:全 0 最好
- 5 鸟健康度:每只鸟 `runs 1 · fails 0 · subs N`

点"**继续 Phase 3 →**"

### Phase 3 · 逐条确认(⭐ 最高频)

**键盘优先**(不用鼠标):

| 键 | 动作 |
|---|---|
| `j` | 下一条 |
| `k` | 上一条 |
| `y` | 接受当前条 |
| `n` | 驳回当前条 |
| `e` | 编辑当前条 · 进 textarea 改 |

每条评审看这几个视觉元素:
- **鸟头像 + 职能徽章**:哪只鸟提的
- **苍鹰验证徽章**:通过 ✓ / 撤回 ⊖ / 补充 ＋
- **依据区**:原文引用 + 验证状态(已验证 ✓ / 验证失败 ✗ / 未验证 ⊖)
- **mono 元数据**:`model=sonnet-4-6 conf=0.82 tokens=2.1k rule=R042`

**注意**:
- 依据验证失败的条目**默认折叠** · 要主动展开才能 accept
- 置信度 < 0.7 的条目会视觉弱化(暖色)
- 底部有"**➕ 我发现一个他们漏掉的问题**"· 漏报反馈按钮(试试看)

走完所有条 · 点"**生成报告 →**"

### Phase 4 · 报告

- **顶部元信息卡**:reviewer / workspace / mode / 总条目 / 接受率 / 啄伤度
- **反馈回声 banner**(v2 才真接数据 · 现在是占位文案)
- **按维度分组的摘要**:每个维度一个卡片 + BirdAvatar + decision chip ✓✗✎
- **底部 3 导出按钮**:
  - **下载 md**:浏览器直接下载文件
  - **保存到 wiki**:存到 workspace 的 wiki 目录(内测期间可能失败 · 没接真环境)
  - **推送飞书**:需要 FEISHU env 配好才能用(内测可能失败)

## 3 份 sample PRD

在 `samples/pm-preview/` 下 · 你可以直接拖到 Phase 0 上传框:

| 文件 | 长度 | 场景 | 测什么 |
|---|---|---|---|
| `sample-1-favorites.md` | 500 字 | 企业收藏功能 | 快速跑一次 · 熟悉流程 |
| `sample-2-risk-alert.md` | 1500 字 | 风险告警订阅 | 看 4 维度 worker 怎么分工 |
| `sample-3-entity-graph.md` | 3500 字 | 企业关联图谱 | 完整场景 · 看苍鹰交叉校验 |

每份 PRD 都故意**埋了可评审点**:
- 业务目标没写具体数字(业务鸟应该挑)
- 字段跨表口径冲突(数据鸟应该挑)
- 文案歧义 / 异常流缺失(体验鸟应该挑)
- 下游依赖 SLA 未声明(风险鸟应该挑)

**试用建议**:sample-1 先跑一次熟悉界面 · 再用 sample-3 看完整体验。

## Sprint 5 v2 预留(内测期间**不用管**)

TopBanner 右上角有 **Runs** 和 **System** 入口 · 点进去是 **sample 数据 + 黄色警示 banner**:

- `/runs/diff` · 对比两次 run(baseline vs shadow)
- `/runs/:id/replay` · 回放某次 run 的完整 event 流
- `/system/health` · consistency 趋势 + rule 权重 Top 6 + eval 基线
- `/system/prompts` · 5 鸟的 prompt 摘要 + 激活 rule 集

这些是 **v2 接真实数据的预留框架** · 内测期间可以点开看看 UI · 但数据是假的,不代表真实系统状态。

## 反馈 5 个问题

内测第 3 天 · 麻烦花 2 分钟填一下(微信发或回群):

1. **气质像什么?** Linear / 飞书文档 / 还是像杂志(Kinfolk)?

2. **Phase 3 键盘 j/k/y/n/e 顺手吗?** 1-5 分 · 简单说为什么

3. **评审质量如何?** sample PRD 里故意埋的那些问题(业务目标缺数字 / 字段口径冲突 等)· 4 鸟抓到了吗?苍鹰交叉校验有价值吗?

4. **导出报告格式能用吗?** 你会直接拿这个报告发给研发吗?需要改什么?

5. **最想改一个地方是?** 不限定 · 一句话。

## 数据风险说明

内测期间建议**只用 sample PRD**。真实业务 PRD 别上传,因为:
- 内测走 Cloudflare Tunnel · 明文经过 Cloudflare 边缘节点(TLS 但 CF 理论能读)
- 评审逻辑会把 PRD 内容传给 Anthropic API(美国服务器)
- 内测环境部署在开发机 · 没有内网 VM 级别的合规保障

正式上线前会走公司 IT 申请内网 VM · 到时候再碰真数据。
