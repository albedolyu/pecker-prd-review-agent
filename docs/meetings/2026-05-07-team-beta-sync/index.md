# 啄木鸟内部团队版会议材料

会议建议目标：把啄木鸟从“个人本地工具”推进到“受控内部 Beta 试用”。

建议使用顺序：

1. `00_meeting_brief.md`：开场一页，对齐结论、证据、风险和需要领导拍板的事项。
2. `01_demo_talk_track.md`：现场演示讲稿，按 8-10 分钟 UI 流程走。
3. `02_decisions_and_rollout.md`：会中需要定下来的决策、试用节奏和验收口径。
4. `03_pm_qa.md`：产品同事代表可能会问的问题和建议回答。
5. `04_after_meeting_followup.md`：会后执行清单。

当前建议口径：

啄木鸟可以进入小范围内部 Beta，但不建议直接作为公司级正式系统开放。第一阶段只给 2-3 位 PM 代表试用脱敏 PRD，跑通反馈闭环、成本闸和服务端部署后，再扩到 5-6 人并发使用。
