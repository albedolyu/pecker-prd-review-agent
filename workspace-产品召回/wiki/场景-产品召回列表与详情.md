# 场景-产品召回列表与详情

## 用户场景

### Web 端
- **入口**：企业详情页 → 经营风险模块 → 公示催告下方
- **列表页**：展示企业维度的召回记录，默认按发布日期从晚到早排序，支持切换排序
- **详情弹窗**：点击列表行展开详情，展示公告正文（content 字段）

### APP 端
- **入口**：企业详情页（具体位置参见 Figma 原型）
- **列表页**：与 web 端一致的字段和排序规则
- **详情页**：独立页面（非弹窗）展示公告全文

## 展示字段

| 视图 | 展示字段 |
|------|---------|
| 列表页 | 发布日期、召回产品、召回原因、实施单位、召回数量、信息来源、公告内容（截断+展开收起） |
| 详情弹窗/页 | 同上 + 公告正文（content）完整展示 |

## 关键交互规则

- 召回产品、召回原因超过两行时显示展开/收起（具体行数定义待明确，见 R-012）
- 空值字段显示"-"（完整规则见 R-007，原 PRD 有内容截断）
- NEW 标签展示规则待明确（见 R-015）

## 数据过滤规则

- `data_status = 0`（全端通用）
- `riskbird_status = 0`（风鸟端特有，字段定义待补充，见 R-001）
- 两条件的布尔关系待明确（见 R-004）

## ⚠️ 知识盲区

- riskbird_status 字段未在 DDL 中定义，含义不明
- 四态 UI（加载/失败/筛选空/数据空）在 PRD 中未定义
- NEW 标签生命周期规则未定义
