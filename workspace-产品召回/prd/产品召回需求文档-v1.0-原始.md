## 1. 业务上下文 (Business Context)

### 1.1 需求来源

竞品企查查有该维度数据，且as与智慧星光数据源有该类型数据，因此在风鸟上补充该数据丰富数据类型。

----

### 1.2 核心价值

- **目标用户**：
  - 商业合作：需要深入了解企业是否存在产品召回，为采购投资决策提供支持

- **业务价值**：
  - 价值1：丰富风鸟平台数据类型，为用户提供更全面的企业经营风险信息。

---

### 1.3 数据介绍

#### 1.3.1 数据背景

- 产品召回起因：消费者投诉举报、企业内部质量追溯系统或政府主管部门监督检査等途径而被发现

- 产品召回报送：生产者应当自确认消费品存在缺陷之日起5个工作日内或被责令召回之日起5个工作日内通过信息系统向所在地省级质检部门备案召回计划。省级质检部门应当在5个工作日将生产者备案的召回计划通过信息系统报送质检总局。

- 产品召回发布：生产者所在地省级质检部门应当及时向社会公布生产者已经确认并备案的缺陷消费品信息以及生产者实施召回的相关信息。质检总局责令召回以及受质检总局缺陷调查影响实施召回的，由质检总局负责向社会公布相关信息。

#### 1.3.2 监管部门及查询渠道

| 产品类别      | 监管部门                                    | 查询渠道                                   |
| ------------- | ------------------------------------------- | ------------------------------------------ |
| 消费品        | 国家市场监督管理总局及地方各级市场监管局    | 市场监管总局网站、市场监管总局召回中心网站 |
| 汽车产品      | 国家市场监督管理总局 (下设缺陷产品管理中心) | 市场监管总局缺陷产品管理中心官网           |
| 药品/医疗器械 | 国家药品监督管理局                          | 国家药监局官网                             |
| 食品          | 国家市场监督管理总局及地方各级市场监管局    | 市场监管总局网站、市场监管总局召回中心网站 |

​         目前明确有专门召回程序的产品例如消费品，汽车产品、食品、药品、医疗器械等，应分别按照《缺陷消费品召回管理办法》《缺陷汽车产品召回管理条例》《食品召回管理办法》《药品召回管理办法》《医疗器械召回管理办法》等进行召回

### 2.1 物理表定义

#### 主表：`ds_risk_recall`

```sql
CREATE TABLE `ds_risk_recall` (
  `id` bigint(20) NOT NULL AUTO_INCREMENT COMMENT '自增id',
  `title` varchar(500) DEFAULT NULL COMMENT '公告标题',
  `rdate` date DEFAULT NULL COMMENT '发布时间',
  `content` text COMMENT '公告正文',
  `content_html` text COMMENT '公告正文_html',
  `entid` bigint(20) DEFAULT NULL COMMENT '企业ID',
  `entname` varchar(200) DEFAULT NULL COMMENT '企业名称',
  `rc_product` varchar(500) DEFAULT NULL COMMENT '召回产品',
  `rc_reason` text COMMENT '召回原因',
  `rc_num` varchar(20) DEFAULT NULL COMMENT '召回数量',
  `source` varchar(200) DEFAULT NULL COMMENT '信息来源',
  `url` varchar(800) DEFAULT NULL COMMENT '来源url',
  `df_id` bigint(20) DEFAULT NULL COMMENT '源表对应id',
  `data_status` tinyint(2) NOT NULL DEFAULT '0' COMMENT '数据状态字段（0有效，1历史，2删除，3去重）',
  `unique_id` varchar(32) NOT NULL COMMENT '业务唯一键',
  `create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '入库时间',
  `update_time` datetime DEFAULT NULL COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_df_id` (`df_id`),
  KEY `idx_entid` (`entid`),
  KEY `idx_unique_id` (`unique_id`),
  KEY `idx_update_time` (`update_time`)
) ENGINE=InnoDB AUTO_INCREMENT=26855 DEFAULT CHARSET=utf8mb4 COMMENT='产品召回';
```

## 3. 业务规则 (Business Rules)

### 3.1 数据查询规则

#### 3.1.1 通用规则

| 规则项 | 规则说明 |
|-------|----------|
| 查询入口 | 从ds_risk_recall表查询 |
| 企业过滤 | entid = 查询企业的entid |
| 数据状态过滤 | 只查询 data_status = 0 |

#### 3.1.2 个性化规则

| 数据类型 | 规则项   | 规则说明              |
| -------- | -------- | --------------------- |
| 产品召回 | 默认排序 | 按照rdate从晚到早排序 |

---

### 3.2 召回产品&召回原因展示规则

**展示结构**：

- 召回产品超过两行增加展开收起功能
- 召回原因超过两行增加展开收起功能

---

### 3.3 时间格式规则

| 字段 | 展示格式 | 说明 |
|-----|---------|------|
| rdate | yyyy-MM-dd | 如：2025-11-28 |

---

### 3.4 风鸟端特殊规则

- 只显示riskbird_status = 0的数据

----

### 3.5 排序规则

| 数据类型 | 支持手动排序的字段                                           |
| -------- | ------------------------------------------------------------ |
| 产品召回 | 发布日期，默认按照发布日期从晚到早排序，支持切换到按照发布日期从早到晚排序 |
