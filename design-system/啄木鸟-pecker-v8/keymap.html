冲突检查

✓ 无冲突 j/k 单键

仅在 comment-list focus 时生效。文档中用 Shift + J/K 避开列表。

✓ 无冲突 1..5 复用

Phase 2 / Phase 3 同时用 1..5，但两个 Phase 互斥（不会同屏），不冲突。

✓ 无冲突 r 单键

Phase 1.5 / Phase 2 都用 r 重跑，语义一致，不冲突。

⚠ 注意 g 作 Leader

g 作为 leader 等待 200ms 第二键；单独 g 不触发跳转，避免与 Gmail 类习惯冲突。

⚠ 注意 / 与浏览器

Firefox 默认 / 是"快速查找"，我们在 DocumentView 内 preventDefault 后使用。

✓ 无冲突 ⌘+K

命令面板用 ⌘+K（Linear / Notion 共识），不与浏览器冲突。

ℹ 说明 输入框豁免

所有单键在 input / textarea / contenteditable 获得焦点时不触发。

ℹ 说明 ShortcutHint 显示策略

悬停 600ms 后显示，或开启"始终显示"偏好。不要在 hover 0ms 弹出，噪声太大。
