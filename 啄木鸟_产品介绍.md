# 啄木鸟 (Pecker) — PRD 评审 Agent

基于 **OpenAI 兼容 API key + model_router** 构建的 PRD 结构化评审工具；团队版默认走 GPT 路由，避免多人并发挤占个人 CLI/OAT session。啄木鸟携手评审团（业务鸟、数据鸟、体验鸟、风险鸟）并行评审 PRD，苍鹰交叉校验去误报补漏报，杜鹃验证评审质量，鸮鹦整理知识库，信鸽从下游 AI Coding 结果 + Web 端用户决策中收集反馈持续改进，伯劳在 push 前审核产出质量。

> **项目状态**: 自动生成的 `STATUS.md`（`scripts/generate_status.py` 从 git log / pytest / session 事件生成），替代早期手写自评。不再维护 harness 自评分。

## 它做什么

把「PM 拿着 checklist 逐条检查 PRD」这件事交给啄木鸟，但保留人的判断权：

1. **知识入库** — 啄木鸟阅读业务资料（PDF / Excel / Markdown / 图片），提取关键信息写入 Obsidian 格式知识库
2. **并行结构化评审** — 四只评审鸟各司其职，并行检查 PRD 的结构、质量、AI Coding 友好度、数据质量
3. **交叉校验** — 苍鹰对四鸟合并结果做二次审查：误报检测、漏报补充、冲突调解
4. **依据硬验证** — 每条改进项的依据必须在知识库或评审规则中找到实证，找不到的自动撤回
5. **交互确认** — 每条改进项附依据和严重度，使用者逐条 Y/N/E 确认，啄木鸟会审核反馈依据是否充分
6. **报告输出** — 生成改动报告、差异报告、交互记录，评审发现反哺知识库
7. **质量门禁** — 伯劳在 push 前自动检查安全红线、wiki 一致性和报告完整性
8. **反馈闭环** — 信鸽从 AI Coding 结果中采集信号，自动更新规则权重、提出新规则提案
9. **Web 版** — Next.js + FastAPI 全栈实现（Streamlit 版作为迁移期 fallback 保留），同事无需安装 CLI，浏览器打开即可评审，JWT 鉴权 + 多人并发 + 断点草稿恢复

## 核心设计原则

- **不编造依据** — 每条改动必须有可回溯来源（知识库 / 评审规则 / 外部参考），找不到依据不提
- **不擅自改 PRD** — 只输出建议，使用者决定是否采纳
- **知识持续累积** — 知识库是累积编译产物，每次评审让它更丰富，而非从头整理
- **C 类改动必须标记** — 竞品/行业惯例不能作为确定性依据，标为「待确定项」
- **下游反馈驱动进化** — 评审规则的有效性由 AI Coding 结果 + 用户 Y/N/E 决策共同验证，无效规则 impact_score 自动降权、漏报模式生成新规则
- **Worker 不互调** — orchestrator → specialist workers → meta-reviewer 单向拓扑，Worker 之间只读共享 scratchpad，不相互影响输出
- **meta-reviewer 只审不重审** — 苍鹰的漏报补充硬上限 2 条（schema maxItems + parser 双保险），苍鹰自己的判断再由 `goshawk_performance_history.json` 做反向校验

## 关键特性

### Prompt Caching
系统提示词和 PRD 内容设置 `cache_control`，缓存命中后只收 10% 费用。10 轮评审省约 ~60% token 费用。

### Microcompact 上下文管理
借鉴 Claude Code 的策略，2 轮后自动清理旧的 tool_result 内容（替换为占位符），保留消息结构。PRD 原文放入 system prompt 享受缓存，不受清理影响。

### Scratchpad 状态板
维护 `_scratchpad.md` 文件跟踪评审进度（当前 Phase、改进项状态、Wiki 变更、知识盲区），即使上下文被压缩 agent 也不会丢失全局状态。

### 权限围栏
`raw/` 和 `prd/` 只读，`wiki/` 和 `output/` 可写，bash 命令有白名单，git push 需用户确认。

### Session 断点恢复
每轮对话写入 JSONL 文件，脚本崩溃后可恢复未完成的评审会话。

### 多人隔离
支持 git worktree，多人同时评审不同 PRD 互不干扰。

## 快速开始

```bash
pip install -r requirements.txt

# 在 .env 中配置（团队版默认 OpenAI 兼容 API）
OPENAI_API_KEY=<服务端真实 key>
OPENAI_BASE_URL=https://pikachu.claudecode.love/v1
OPENAI_WIRE_API=responses
WIKI_PATH=/path/to/prd-wiki
REVIEWER=你的名字

# 启动评审
python run_session.py PRD名称 --workspace /path/to/workspace

# Web 版（推荐给非技术同事）
streamlit run legacy/app.py
# 浏览器打开 http://localhost:8501

# AI Coding 完成后，运行信鸽收集反馈
python feedback.py \
  --code-dir /path/to/code \
  --report output/PRD_改动报告_xxx.md \
  --prd prd/xxx.md \
  --scope keyword1,keyword2
```

详细操作见 [啄木鸟_使用指南.md](./啄木鸟_使用指南.md)。
