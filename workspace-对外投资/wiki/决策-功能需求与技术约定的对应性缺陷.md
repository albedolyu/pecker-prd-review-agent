# 功能需求与技术约定的对应性缺陷

功能需求列表（FR-01~FR-N）定义了多条后端接口需求，但「技术约定」节仅给出 API 基础路径和鉴权方式，未覆盖各 FR 对应的接口路径、请求/响应格式、错误码等。检查点：对每条 FR，确认是否在技术约定中有对应的接口定义。规则编号：RC-005。常见于业务流程复杂、跨多个接口的 PRD。
