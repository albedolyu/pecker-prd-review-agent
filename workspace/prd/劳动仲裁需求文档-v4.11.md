## 1. 业务上下文 (Business Context)

### 1.1 需求来源

25年12月的工行招标要求劳动仲裁数据，信数正好借此机会补充劳动仲裁数据以满足工行招标要求，同时在风鸟上补充该数据丰富数据类型。

----

### 1.2 核心价值

- **目标用户**：
  - 求职应聘：求职人员发现目标公司存在劳动仲裁案件，可能不会投递简历或入职。
  - 商业合作：通过劳动仲裁数据侧面判断企业的稳定性。若劳动仲裁数据很多，则间接表明公司人员流动性大，公司不一定稳定。

- **业务价值**：
  - 价值1：满足工商银行2025年12月招标需求，提供劳动仲裁数据。
  - 价值2：丰富风鸟平台数据类型，为用户提供更全面的企业经营风险信息。

---

### 1.3 数据介绍

#### 1.3.1 法规/政策依据

- 普通企业：《中华人民共和国劳动争议调解仲裁法》
- 事业单位：《事业单位人事管理条例》《中华人民共和国公务员法》
- 各个地方有自己的补充条例
  -  地方立法与国家法律的从属、补充与细化关系
  - 遵循 “不抵触、有补充” 原则
  - 条例举例：《浙江省劳动人事争议调解仲裁条例》,《广西壮族自治区劳动人事争议调解仲裁条例》,《福建省事业单位人事争议处理规定》,《鞍山市劳动争议调解条例》

#### 1.3.2 相关流程

| 步骤                    | 内容                                                         |
| ----------------------- | ------------------------------------------------------------ |
| step1：<br>发生劳动争议 | 《中华人民共和国劳动争议调解仲裁法》第一章第二条规定，以下内容属于劳动争议的情形：<br />1.因确认劳动关系发生的争议；<br/>2.因订立、履行、变更、解除和终止劳动合同发生的争议；<br/>3.因除名、辞退和辞职、离职发生的争议；<br/>4.因工作时间、休息休假、社会保险、福利、培训以及劳动保护发生的争议；<br/>5.因劳动报酬、工伤医疗费、经济补偿或者赔偿金等发生的争议；<br/>6.法律、法规规定的其他劳动争议。 |
| step2：<br>当事人申请   | 当事人递交劳动仲裁申请书及相关材料、证据。                   |
| step3：<br>审查受理     | 劳动争议仲裁委员会在收到仲裁申请之日起五个工作日内作出受理或者不予受理的决定。 |
| step4：<br>仲裁准备     | 仲裁委员会对决定受理的劳动争议案件应当自立案之日起七日内组成仲裁庭。 |
| step5：<br>仲裁审理     | 《中华人民共和国劳动争议调解仲裁法》第三章第二十六条规定：劳动争议仲裁公开进行，但当事人协议不公开进行的或者涉及国家秘密、商业秘密和个人隐私的除外。 |

#### 1.3.3 相关网站

* 以上海和深圳为例，两个城市都会在市级和各个区级的政府网站的人社版块上公布劳动仲裁数据，各个网站数据彼此独立不重复，可以从案号中的地区代字确定不重复。

* 如果当地人力资源和社会保障局有自己的独立的官网，则可能在独立的官网上公布劳动仲裁数据。

#### 1.3.4 数据采集

1. 数据源
   1. 增量：智慧星光提供文本内容，信数结构化
   2. 存量-智慧星光已采集：智慧星光提供文本内容，信数结构化
   3. 存量-智慧星光未采集：信数采集文本内容，信数结构化

2. 采集方案
   1. 从正文中解析：申请人、被申请人、仲裁机构、公告时间、具体仲裁原因（所有仲裁公告都可以用“劳动争议”来概述）、开庭时间等
   2. 目前能确定大语言模型可以解析，之后从成本角度出发请开发评估是否能通过正则或其他更具性价比的方案。
3. 根据调研，劳动仲裁分类两类，一类是送达公告，还有一类包含开庭时间等信息的开庭公告
   1. 送达公告样例：https://www.shyp.gov.cn/shypq/zmhd-zcgg/20251210/496052.html
   2. 开庭公告样例：https://www.jinshan.gov.cn/zhengwu/rsj-ldgx/2025/338/6346.html
4. 已知的公布劳动仲裁数据的网站：[钉钉文档](https://alidocs.dingtalk.com/i/nodes/dQPGYqjpJYgg0x22fRjMNEQMWakx1Z5N?utm_scene=person_space&iframeQuery=sheet_range%3Dkgqie6hm_14_3_1_1)

### 1.4 竞品分析

**1.4.1 企查查**：

1. 数据归类：经营风险大类
2. 企业主页

```
将劳动仲裁分为“开庭公告”和“送达公告”
1、送达公告（eg:恒大集团有限公司）
-- 列表页：案号、申请人、被申请人、发布日期
-- 详情页：列表页字段+原内容
2、开庭公告（eg:郑州派尼服装有限公司）
-- 列表页：案号、申请人、被申请人、主案由、发布机构、开庭日期（非时间）
-- 详情页：列表页字段+仲裁员+书记员+仲裁地点
```

3. 查风险无，司法大数据无，仅信用大数据有。

**1.4.2 天眼查**：

1. 放在经营风险大类中
2. 有“送达公告”分类（阿里巴巴（中国）有限公司），“开庭公告”分类未找到样例
3. 送达公告字段和企查查一致

## 3. 业务规则 (Business Rules)

### 3.1 数据查询规则

#### 3.1.1 通用规则

| 规则项 | 规则说明 |
|-------|----------|
| 查询入口 | 从ds_risk_labour_arbitration_ent表查询 |
| 企业过滤 | entity_id = 查询企业的entid |
| 主体类型过滤 | entity_type = 2（只查企业） |
| 数据状态过滤 | data_status = 0（只查有效数据）主表和ent表都要符合本规则 |
| 公告类型过滤 | type = 1（送达公告）或 type = 2（开庭公告） |
| 风鸟端过滤 | riskbird_status=0需要展示，为1不用展示 |

#### 3.1.2 个性化规则

| 数据类型          | 规则项   | 规则说明                     |
| ----------------- | -------- | ---------------------------- |
| 劳动仲裁-送达公告 | 默认排序 | 按照publish_date从晚到早排序 |
| 劳动仲裁-开庭公告 | 默认排序 | 按照open_time从晚到早排序    |

---

### 3.2 当事人展示规则

**展示结构**：

- 当事人按立场分组展示：申请人一组、被申请人一组
- 每组内可包含多个当事人（个人或企业）
- 多个当事人用顿号（、）分隔

**数据来源**：
- 根据case_code和table_id查询ent表
- 按standpoint字段分组
  - standpoint=1：申请人
  - standpoint=2：被申请人

**特殊情况说明**：
- 申请人多为个人，但也可能是企业
- 被申请人多为企业，但也可能是个人

---

### 3.3 脱敏规则

| 字段场景 | 脱敏规则 | 示例 |
|---------|---------|------|
| 申请人/被申请人中的自然人 | 姓+** | 张三 → 张** |
| 申请人/被申请人中的企业 | 不脱敏 | 上海ABC公司 → 上海ABC公司 |
| 仲裁员 | 不脱敏 | 张三 → 张三 |
| 书记员 | 不脱敏 | 李四 → 李四 |
| 公告内容中的人名 | 姓+** | 与上述规则一致 |
| 公告内容中的企业名 | 不脱敏 | 保持原样 |

**脱敏目的**：
- 保护自然人隐私
- 企业信息公开透明，不脱敏
- 仲裁员、书记员为公务人员，不脱敏

---

### 3.4 时间格式规则

### 3.6 筛选规则

| 模块 | 筛选维度 | 规则说明 |
|------|---------|:---------|
| 送达公告 | 仲裁机构 | 首先将主表.area_code映射到对应的行政区划，然后根据行政区划分为三级筛选或两级筛选。<br />若映射后的行政区划是“省级行政区及其下属地区”，例如江苏省南京市“则为三级筛选，第一级为”省级行政区“、第二级为”地级市“、第三级为对应地区的机构列表及数量。<br>若映射后的行政区划是”直辖市及其下属区县“，例如北京市朝阳区，则分两级筛选，第一级为”直辖市“，第二级为对应地区的机构列表及数量。<br />筛选项按照主表.area_code从小到大排序 |

- 数量统计：每个筛选项需返回该维度下的记录数量
- 调用时机：仅当请求参数指定需要筛选条件时才返回（避免不必要的聚合查询）
- 按需返回：非首次加载或用户点击筛选时才请求筛选条件数据

### 3.8 搜索框搜索规则

劳动仲裁的送达公告和开庭公告的搜索框均支持搜索以下4个字段

| 字段              | 搜索模式               | 备注                                               |
| ----------------- | ---------------------- | -------------------------------------------------- |
| 案号              | 支持精确搜索、模糊搜索 | -                                                  |
| 案由              | 支持精确搜索、模糊搜索 | -                                                  |
| 当事人            | 支持精确搜索、模糊搜索 | 分为申请人和被申请人                               |
| 仲裁机构/发布机构 | 支持精确搜索、模糊搜索 | 送达公告对应：仲裁机构<br />开庭公告对应：发布机构 |
