## 1. 业务上下文 (Business Context)

### 1.1 需求来源

很多客诉提及，希望我们能够新增纳税人资质数据维度。

----

### 1.2 核心价值

- **目标用户**：
  - 任何想要了解某企业的纳税情况，以及通过纳税情况推测企业经营实力的用户。
  
- **业务价值**：
  - 价值1：丰富风鸟平台数据类型，为用户提供更全面的企业经营信息。

---

### 1.3 数据介绍

#### 1.3.1 业务知识

- 新企业注册后，默认成为**小规模纳税人**。
- 年应税销售额未超 500 万元，属于**小规模纳税人**。
  - 满足以下条件，可申请转为一般纳税人：会计核算健全，能按照国家统一会计制度设置账簿，还可提供准确税务资料，可主动向主管税务机关申请登记为**一般纳税人**。
- 年应税销售额超过 500 万元，此时企业通常需强制登记为**一般纳税人**。

#### 1.3.2 数据采集

- 通过各地区的电子税务局，输入纳税人名称及纳税人识别号信息，查询得到纳税人资格类型

#### 1.3.3 数据现状

- 数据来源：从as接入纳税人状态类别表（bn_ministry.tax_state），该表已接入xs数据库，ds_company_tax_state。

- 数据质量探查：

  1. 一家企业只有一条纳税人资质数据

  2. 空值率：

     总量：39782925

     主管税务机关：空值数据量1357183，比例3.41%，查询电子税务局是正常情况

     认定有效期起：空值数据量21253177，比例53.24%，空值率较高，但因为是官方字段，先做保留。部分数据查询电子税务局无数据，企查查是正常情况

     认定有效期止：空值数据量21871903，比例54.98%，可接受为空

### 2.1 物理表定义

#### 主表：`ds_company_tax_state`

```sql
CREATE TABLE `ds_company_tax_state` (
  `id` bigint(20) NOT NULL AUTO_INCREMENT,
  `entid` bigint(20) NOT NULL COMMENT '企业UID',
  `entname` varchar(200) DEFAULT NULL,
  `uniscid` varchar(30) DEFAULT NULL,
  `taxid` varchar(30) DEFAULT NULL COMMENT '纳税人识别码',
  `stype` varchar(10) DEFAULT NULL COMMENT '纳税人类型（0-小规模纳税人，1-按适用税率征收增值税小规模纳税人，2-一般纳税人，3-简易办法征收一般纳税人，4-出口退（免）税企业，5-小型微利企业，6-增值税汇总纳税企业）',
  `tstatus` varchar(10) DEFAULT NULL COMMENT '状态:1-正常，0-不正常',
  `pdate` date DEFAULT NULL COMMENT '决定日期',
  `sdate` date DEFAULT NULL COMMENT '开始日期',
  `edate` date DEFAULT NULL COMMENT '结束日期',
  `taxorg` varchar(150) DEFAULT NULL COMMENT '管理税务机关',
  `tax_manager` varchar(150) DEFAULT NULL COMMENT 'COMMENT税务负责人',
  `url` text,
  `sp_id` bigint(20) NOT NULL COMMENT '源数据ID',
  `data_status` int(11) NOT NULL DEFAULT '0' COMMENT '数据状态字段（0有效，1历史，2删除，3去重）',
  `create_time` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` timestamp NULL DEFAULT NULL COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_sp_id` (`sp_id`),
  KEY `idx_entid` (`entid`),
  KEY `idx_update_time` (`update_time`)
) ENGINE=InnoDB AUTO_INCREMENT=40364063 DEFAULT CHARSET=utf8 COMMENT='税务局-纳税人状态类别';
```
