# 〇、业务上下文

## 1 需求来源

25年12月的工行招标要求劳动仲裁数据，信数正好借此机会补充劳动仲裁数据以满足工行招标要求，同时在风鸟上补充该数据丰富数据类型。

----

## 2 核心价值

- **目标用户**：
  - 求职应聘：求职人员发现目标公司存在劳动仲裁案件，可能不会投递简历或入职。
  - 商业合作：通过劳动仲裁数据侧面判断企业的稳定性。若劳动仲裁数据很多，则间接表明公司人员流动性大，公司不一定稳定。

- **业务价值**：
  - 价值1：满足工商银行2025年12月招标需求，提供劳动仲裁数据。
  - 价值2：丰富风鸟平台数据类型，为用户提供更全面的企业经营风险信息。

---

## 3 数据介绍

### 3.1 法规/政策依据

* 普通企业：《中华人民共和国劳动争议调解仲裁法》

* 事业单位：《事业单位人事管理条例》《中华人民共和国公务员法》

- 各个地方有自己的补充条例
  -  地方立法与国家法律的从属、补充与细化关系
  -  遵循 "不抵触、有补充" 原则
  -  条例举例：《浙江省劳动人事争议调解仲裁条例》,《广西壮族自治区劳动人事争议调解仲裁条例》,《福建省事业单位人事争议处理规定》,《鞍山市劳动争议调解条例》

### 3.2 相关流程

| 步骤                    | 内容                                                         |
| ----------------------- | ------------------------------------------------------------ |
| step1：<br>发生劳动争议 | 《中华人民共和国劳动争议调解仲裁法》第一章第二条规定，以下内容属于劳动争议的情形：<br />1.因确认劳动关系发生的争议；<br/>2.因订立、履行、变更、解除和终止劳动合同发生的争议；<br/>3.因除名、辞退和辞职、离职发生的争议；<br/>4.因工作时间、休息休假、社会保险、福利、培训以及劳动保护发生的争议；<br/>5.因劳动报酬、工伤医疗费、经济补偿或者赔偿金等发生的争议；<br/>6.法律、法规规定的其他劳动争议。 |
| step2：<br>当事人申请   | 当事人递交劳动仲裁申请书及相关材料、证据。                   |
| step3：<br>审查受理     | 劳动争议仲裁委员会在收到仲裁申请之日起五个工作日内作出受理或者不予受理的决定。 |
| step4：<br>仲裁准备     | 仲裁委员会对决定受理的劳动争议案件应当自立案之日起七日内组成仲裁庭。 |
| step5：<br>仲裁审理     | 《中华人民共和国劳动争议调解仲裁法》第三章第二十六条规定：劳动争议仲裁公开进行，但当事人协议不公开进行的或者涉及国家秘密、商业秘密和个人隐私的除外。 |

### 3.3 相关网站

eg1：国家会在"执行信息公开网"的被执行人模块统一公布被执行人信息，链接：https://zxgk.court.gov.cn/zhixing/。

eg2：国家没有统一官网，各地方会在各自的人社局公布劳动仲裁数据。

### 3.4 数据采集

1. 数据源
   1. 增量：智慧星光提供文本内容，信数结构化
   2. 存量-智慧星光已采集：智慧星光提供文本内容，信数结构化
   3. 存量-智慧星光未采集：信数采集文本内容，信数结构化
2. 采集方案
   1. 从正文中解析：申请人、被申请人、仲裁机构、公告时间、具体仲裁原因（所有仲裁公告都可以用"劳动争议"来概述）、开庭时间等
3. 根据调研，劳动仲裁分为两类，一类是送达公告，还有一类包含开庭时间等信息的开庭公告
   1. 送达公告样例：https://www.shyp.gov.cn/shypq/zmhd-zcgg/20251210/496052.html
   2. 开庭公告样例：https://www.jinshan.gov.cn/zhengwu/rsj-ldgx/2025/338/6346.html
4. 已知的公布劳动仲裁数据的网站：[钉钉文档](https://alidocs.dingtalk.com/i/nodes/dQPGYqjpJYgg0x22fRjMNEQMWakx1Z5N?utm_scene=person_space&iframeQuery=sheet_range%3Dkgqie6hm_14_3_1_1)

# 二、企业主页

## 1. 业务规则

### 1.1 前端字段映射

【劳动仲裁】分为【送达公告】和【开庭公告】两类，通过主表或ent表的type字段区分。

| 前端展示字段        | 数据表来源                       | 说明                                           |
| ------------------- | -------------------------------- | ---------------------------------------------- |
| 区分公告类型        | 主表的type                       | 1:送达公告, 2:开庭公告                         |
| 案号                | ent表.case_code                  | -                                              |
| 当事人列表          | ent表.entity_name, standpoint    | 按standpoint分组展示                           |
| 申请人              | ent表中standpoint=1的entity_name | 多个用顿号分隔                                 |
| 被申请人            | ent表中standpoint=2的entity_name | 多个用顿号分隔                                 |
| 仲裁机构 / 发布机构 | 主表.arbitration_org             | 送达公告展示"仲裁机构"、开庭公告展示"发布机构" |
| 发布日期            | 主表.publish_date                | -                                              |
| 公告内容            | 主表.content                     | 需对人名脱敏、企业名增加跳转链接               |
| 案由                | 主表.case_reason                 | -                                              |
| 仲裁员              | 主表.arbitrator                  | 不打码                                         |
| 书记员              | 主表.recorder                    | 不打码                                         |
| 地点                | 主表.arbitration_site            | -                                              |
| 开庭时间            | 主表.open_time                   | -                                              |

> **公告内容跳转链接说明**：公告内容（主表.content）中识别到的企业名，通过 `ent表.entity_id`（entity_type=2）关联企业主页。若 entity_id 为空或查询不到对应企业，企业名以纯文本展示，不加链接。

### 1.2 数据查询规则

#### 1.2.1 通用规则

| 规则项       | 规则说明                                                 |
| ------------ | -------------------------------------------------------- |
| 查询入口     | 从ds_risk_labour_arbitration_ent表查询                   |
| 企业过滤     | entity_id = 查询企业的entid                              |
| 主体类型过滤 | entity_type = 2（只查企业）                              |
| 数据状态过滤 | data_status = 0（只查有效数据）主表和ent表都要符合本规则 |
| 公告类型过滤 | type = 1（送达公告）或 type = 2（开庭公告）              |
| 风鸟端过滤   | riskbird_status=0需要展示，为1不用展示                   |

#### 1.2.2 个性化规则

| 数据类型              | 规则项   | 规则说明                     |
| --------------------- | -------- | ---------------------------- |
| 【劳动仲裁-送达公告】 | 默认排序 | 按照publish_date从晚到早排序 |
| 【劳动仲裁-开庭公告】 | 默认排序 | 按照open_time从晚到早排序    |

### 1.3 当事人展示规则

* **展示结构**：
  * 当事人按立场分组展示：申请人一组、被申请人一组
  * 每组内可包含多个当事人（个人或企业）
  * 多个当事人用顿号（、）分隔

* **数据来源**：
  * 根据case_code和table_id查询ent表

### 2.5 API 接口规范

> 接口鉴权统一使用 `Authorization: Bearer {token}`（见技术约定），以下接口均需鉴权。

#### 2.5.0 通用响应规范

**统一响应结构**（2.5.1 - 2.5.4 所有接口的最外层）：

```
{
  "code": int,        // 业务状态码：0=成功，非0=失败
  "message": string,  // 人类可读消息：成功为"ok"，失败为具体错误描述
  "data": object      // 业务数据：成功时为下文 2.5.1-2.5.4 各接口"出参"定义；失败时为 null
}
```

**HTTP 状态码与业务状态码映射**：

| HTTP 状态码 | code   | message 示例                          | 触发场景                                    |
|------------|--------|--------------------------------------|--------------------------------------------|
| 200        | 0      | ok                                   | 请求成功                                    |
| 400        | 40001  | 入参校验失败：{field} 不能为空           | 必填参数缺失 / 类型错误 / page < 1         |
| 400        | 40002  | 入参校验失败：page_size 必须为 20/50/100 | page_size 不在允许枚举值                   |
| 400        | 40003  | 入参校验失败：type 必须为 1 或 2         | 公告类型枚举错误（2.5.3 / 2.5.4）          |
| 401        | 40101  | 未登录或 token 已过期                  | Authorization 缺失 / token 无效 / 已过期   |
| 403        | 40301  | 无权限访问该企业数据                    | 当前用户无 entid 对应企业的查询权限         |
| 404        | 40401  | 记录不存在                            | table_id 在数据库中查不到（仅 2.5.4）       |
| 500        | 50001  | 服务内部错误，请稍后重试                | 后端异常 / 依赖服务不可用                   |

**失败响应示例**：

```
HTTP/1.1 401 Unauthorized
{
  "code": 40101,
  "message": "未登录或 token 已过期",
  "data": null
}
```

**约定**：
- 2.5.1 - 2.5.4 各接口"出参"小节描述的字段，均为成功时 `data` 对象内的字段（即 `code=0` 分支）。
- 失败响应一律遵循上表映射，不再在每个接口下重复列出。
- 分页接口（2.5.1 / 2.5.2）当 `total=0` 时，`data.list` 返回空数组 `[]`，不返回 404。

入参：

  arbitration_org  string  选填  仲裁机构筛选（机构名称）

出参：
  total            int     总记录数

    table_id          long    主表ID（详情接口入参）

  arbitration_site  string  仲裁地点（空值返回"-"）
